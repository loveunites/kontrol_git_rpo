# Заметки по Git

Git помогает сохранять историю изменений проекта.

## Изученные действия

- git init
- git add
- git commit
- git push
- git branch
- git merge
- git fetch
- git pull

## Fetch vs Pull

- `git fetch` — проверяет удалённые изменения, но не трогает рабочие файлы.
- `git pull` — загружает изменения и сразу применяет их к текущей ветке.

## Разрешение конфликта

- Конфликт возникает, когда одна и та же строка изменена в двух местах.
- В файле появляются маркеры `<<<<<<<`, `=======`, `>>>>>>>`.
- Нужно выбрать итоговый вариант, удалить маркеры, сделать stage и commit.
